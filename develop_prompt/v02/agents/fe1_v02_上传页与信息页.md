# Agent FE1 · v02 上传页 + 信息页(多简历 / 空状态 / 处理动画 / 报告入口)

## 你的身份与任务
你是审美在线的前端工程师,把 v01 的单一 `ResumePage` 拆成两页并补齐多简历体验:
- **上传页** `/upload`:输入简历 + 提交后的全屏「处理中」动画,解析完成自动跳信息页。
- **信息页** `/`、`/resume/:resumeId`:左侧简历边栏(切换/改名)、摘要、项目卡(进入拷问 + 查看报告),以及无简历时的空状态。

## 改动清单

### A) `pages/UploadPage.tsx`(替换 FE0 占位)
- 复用 `ResumePage` 的 `ComposeView`(文本/PDF 二选一)、`FailedView` 与图标。
- 提交流程:`createResumeText/File` → 拿到 `resume_id` → `usePolling(getResume)` 至 `status!=='parsing'` →(parsed)`setLastResumeId(id)` 并 `navigate('/resume/'+id)`;失败显示 `FailedView` 可重试。
- **处理动画(修复"像卡住",反馈#2)**:**从点击提交那一刻起**(即 `submitting` 为真时,而不仅是 parsing 阶段)就切到一个**全屏、明显在动**的「处理中」视图,覆盖较慢的上传/Doc2X POST 与随后的解析轮询。要有持续动效(如呼吸的品牌/面试官徽标 + 流动的弧线/微粒/渐变扫光),**无文案**。绝不能出现只有按钮 spinner、主体静止的空窗。

### B) `pages/InfoPage.tsx`(替换 FE0 占位)
- **当前简历解析**:`useParams` 取 `resumeId`;无参数时按 `getLastResumeId()`→否则 `listResumes()` 的最新一项→都没有则进入**空状态**。进入某简历时 `setLastResumeId`。
- **左侧边栏**:`listResumes()` 列出历次简历(显示 `name`,`parsing` 项给个动效态);高亮当前;点击 → `navigate('/resume/'+id)`。**改名**:对当前(或每项)简历提供克制的内联编辑(点名字进入可编辑 → 失焦/回车调 `renameResume` → 本地乐观更新,失败回滚)。边栏内或顶部留一个「新建/上传」入口 → `/upload`(与顶栏一致,二者皆可)。
- **主区**:`getResume(currentId)` → 摘要(`summary.headline` + `items` 的 `Chip`)+ 项目卡列表(复用 v01 `ProjectCard` 视觉)。每张项目卡两个动作:
  - 进入拷问(主):`current_interrogation` 非空时点击 → `navigate('/interrogations/'+current_interrogation.id)`(同 v01)。
  - **查看报告(反馈#1)**:次级入口/按钮 → `navigate('/interrogations/'+current_interrogation.id+'/guide')`,**仅当 `current_interrogation.status==='ended'` 时可点**,否则以视觉表达不可用。
- **空状态(反馈#3)**:无任何简历时,呈现优雅的空状态(图形为主、零文案)+ 一个上传按钮 → `/upload`。
- **解析中防御**:若当前简历 `status==='parsing'`,主区用解析骨架/动画(复用 v01 `ParsingView`)。
- 清理:`ResumePage.tsx` 不再被引用,可删除(或抽出可复用片段后删除)。
