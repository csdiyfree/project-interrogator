## 改动清单

### 1) 顶部导航(反馈#1)——改 `InterrogationPage` 的 `<header>`(现约 205–215 行)
- 左侧保留「项目拷问」小标 + 项目名 + 第 N 轮 Chip。
- **新增两个图标按钮**(放在 header 右侧):
  - 「返回信息页」→ `navigate('/resume/' + project.resume_id)`(`project` 来自现有 `getProject`,已含 `resume_id`)。
  - 「查看报告」→ `navigate('/interrogations/' + id + '/guide')`,**仅当 `detail.ended`(或 `detail.status==='ended'`)时可点**,否则以视觉表达禁用。
- **删除**页尾那段 `detail.ended && (...)` 的圆形跳指南按钮(现约 280–293 行)——其职责已被顶部「查看报告」取代。

### 2) 项目切换下拉(反馈#4)
- 在项目名旁加一个小三角(caret)图标,点击展开同简历其他项目列表。
- 数据:用 `project.resume_id` 调 `getResume(resume_id)` 取 `projects[]`(每项含 `current_interrogation`);可在打开下拉时懒加载并缓存。
- 列表每项显示项目名,点击 → `navigate('/interrogations/' + p.current_interrogation.id)`;`current_interrogation` 为空的项目(预处理中)以禁用态或仍允许进入(进入后页面会轮询)。当前项目高亮。
- 下拉样式克制(卡片 + 柔影),点击外部关闭。

### 3) 粘附布局(反馈#5)
- 把「页眉 + 双栏」区域约束在视口高度内:页面根用列向 flex 并占满可用高度(注意 `App.tsx` 全局 `<main>` 有 `pt-8 pb-24`,据此扣减或在本页内自管高度)。
- **对话列**内部滚动(`overflow-y-auto`、`min-h-0`),**输入框固定在对话列底部**始终可见;**手稿列**也独立内部滚动。
- 自动滚动改为滚动**对话列容器**到底(替换现有对 `bottomRef.scrollIntoView` 的页面级滚动依赖),保证最新问答与输入框始终在视口内,用户无需滚动整页。
- 移动端:仍可纵向堆叠,但保证输入框与最新一条可见(可让对话区占主要高度、手稿可折叠/置后)。
