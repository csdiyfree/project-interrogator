# Agent BE1 · v02 简历管理(多简历 + 命名 + 改名)

## 你的身份与任务
你是后端工程师,为「项目拷打器」v02 实现**多简历管理**:简历列表、自动命名、手动改名,并在解析结果里带上名称。

## 前置
**BE0 v02 已完成**(`Resume.name` 列、`ResumeListItem/ResumesList/ResumeRename` schema、`ResumeDetail.name`)。本机库已执行 `ALTER TABLE resumes ADD COLUMN name VARCHAR;` 或已重建。

## 先读(在仓库内)
- `develop_prompt/v02/00_第二轮改动计划.md`(契约增量、命名决策)
- 现有代码:`backend/app/routers/resumes.py`、`backend/app/services/resume_service.py`、`backend/app/schemas.py`、`backend/app/models.py`

## 铁律
- **最小改动**:只在 `resumes.py` 与 `resume_service.py` 内增改;复用现有 `_require_session`、归属校验范式、错误格式(`{"code","message"}`)。
- 命名逻辑集中在一个小函数,别散落。

## 改动清单

### 1) 自动命名(`resume_service.py`)
- **命名规则**:`name = f"{姓名} {上传时间}"`。
  - 姓名:从 `summary_json.items` 里找 `label` 含「姓名」的项取其 `value`;找不到则用 `summary_json.headline`;再没有就用日期。
  - 上传时间:用 resume.created_at 格式化为 `M月D日`(精确到日即可;若需更少重名可带 `HH:mm`)。
  - **同会话去重**:若该 session 下已存在同名简历,追加 ` (1)`、` (2)`…直到唯一。写一个 `_unique_name(db, session_id, base) -> str`。
- **落点**:
  - POST 创建时(`routers/resumes.py`)先给一个**临时名**(如 `f"简历 {M月D日 HH:mm}"`),保证列表立刻有标签。
  - `parse_resume` 解析成功、拿到 `summary_json` 后,计算正式 `name` 并写回 `resume.name`(覆盖临时名)。失败路径不强求命名。

### 2) 详情带名称(`routers/resumes.py` 的 `get_resume`)
`ResumeDetail` 增加 `name=resume.name`(`parsing` 分支也带上临时名)。

### 3) 新端点 `GET /api/resumes`(列表)
- 依赖 `_require_session`;查询该 `session_id` 的全部简历,按 `created_at` 倒序。
- 返回 `schemas.ResumesList`,每项 `ResumeListItem(id, name, status, created_at)`。
- 注意路由顺序:`/resumes`(列表)与 `/resumes/{resume_id}`(详情)并存,FastAPI 可正确区分。

### 4) 新端点 `PATCH /api/resumes/{resume_id}`(改名)
- 依赖 `_require_session` + 归属校验(不属于当前会话 → 404 `not_found`)。
- body `schemas.ResumeRename`;`name.strip()` 为空 → 400 `empty_input`。
- 更新 `resume.name`(直接采用用户输入,不做去重),commit,返回更新后的 `ResumeListItem`。
