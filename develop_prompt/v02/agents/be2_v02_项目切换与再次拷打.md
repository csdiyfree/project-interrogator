# Agent BE2 · v02 项目切换支持 + 再次拷打「装作不知道」

## 你的身份与任务
你是后端工程师,为「项目拷打器」v02 做两处最小改动:
1. 让 `ProjectDetail` 带上 `resume_id`,以支持拷问页在同一份简历的项目间切换。
2. 修正「再次拷打」:面试官应当**心里有数、却表演成一个对上一轮一无所知的全新面试官**,不流露已知历史、不复述既有结论。

## 改动 2:再次拷打「装作不知道」(`services/interrogation_service.py`)
当前 `_prev_summary_block` 把上一轮摘要注入为「# 上一轮拷打回顾(供你参考,避免重复…)」,这会让模型公开延续上轮判断。把它改成「绝密内部情报」框架——模型只在心里用它决定深挖方向,表面上从零开始。**用下面的文案替换 `_prev_summary_block` 中非空分支的返回值**:

```
# 绝密内部情报(仅你可见,候选人看不到,你也绝不能流露)
你其实带着一份关于该候选人这个项目的内部情报。但你**必须从头表演一个第一次接触这个项目、对此前结果一无所知的全新面试官**:
- 仍只凭简历原文形成你真实的【初印象】;手稿与问题里都不得出现「上一轮 / 上次 / 这次又 / 还是」等暴露你知道历史的字眼,更不能复述任何既有结论或评价。
- 这份情报只用于在心里决定「该往哪儿多挖一铲」:对其中点到的薄弱、含糊、存疑处,设计更克制也更致命的追问,但表面上要像是你此刻凭对话亲自发现的疑点。
情报内容:
{prev_summary}
```
(`{prev_summary}` 处拼接真实的 `prev_summary` 文本;空 prev_summary 时仍返回空串,行为不变。)

> 校验:`interrogation.md` 现有正文不需要改(它只在该位置插入 `{{prev_summary_block}}`,且「初印象/仅凭简历原文」的措辞与新框架一致)。若你发现 `interrogation.md` 里有任何"参考上一轮""避免重复"之类与新框架冲突的话术,再做最小删改。
