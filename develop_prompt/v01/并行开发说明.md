# 项目拷打器 · v01 并行开发说明

本目录把整个 v01 拆成 **8 个可独立交付的开发任务**,每个对应 `agents/` 下一份**自包含**提示词。你只需把某份提示词整体发给一个新 agent 即可启动该部分开发;提示词内会指引它去读取本目录的契约与提示词附录。

## 执行顺序与并行关系

```
后端轨道:  BE0 ───┬──▶ BE1
                 ├──▶ BE2
                 └──▶ BE3
前端轨道:  FE0 ───┬──▶ FE1
                 ├──▶ FE2
                 └──▶ FE3
```

- **两条轨道(后端/前端)完全并行**,互不阻塞。前端靠 FE0 内置的 mock 层对着契约独立开发,无需等待后端。
- 每条轨道内:**地基(BE0 / FE0)必须先完成**,因为其余任务会 import / 复用它定义的共享代码与约定。地基完成后,3 个模块任务可同时并行。
- 唯一跨轨道耦合点是 [`01_接口契约.md`](01_接口契约.md),已冻结。

## 推荐启动方式

1. 先并行启动 **BE0** 与 **FE0**(两个 agent 同时跑)。
2. BE0 完成后,并行启动 **BE1 / BE2 / BE3**。
3. FE0 完成后,并行启动 **FE1 / FE2 / FE3**。
4. 后端整体就绪后,前端把 `VITE_USE_MOCK` 置为 `false` 做真实联调。

## 验收基线(每个 agent 内有更细的验收清单)

- 后端:`uvicorn app.main:app` 可启动;`/docs` 能看到契约中的全部接口;关键流程(解析→预处理→拷问一轮→结束→指南)可端到端跑通。
- 前端:`npm run dev` 可启动;三页在 `VITE_USE_MOCK=true` 下完全可用;切真实后端后联调通过。
- 共同:不引入契约之外的接口/字段;遵守 [`00_架构设计总览.md`](00_架构设计总览.md) §2 设计铁律。
