## 1. 产品一句话

输入简历 → 系统解析出每个项目条目 → 针对单个项目做「逐项深入追问 + 实时面试官手稿」→ 追问结束后产出「红黄绿灯 + 可勾选改进 todo + 再次拷打」的改进指南。用户可对同一项目反复迭代。

## 2. 设计铁律(来自 `ideas/代码风格要求和契约.md`,所有 agent 必须遵守)

1. **最短实现,严禁过度设计**。本项目绝大部分逻辑就是「拼提示词 → 调 LLM → 拆分结果」。不要引入用不到的抽象、设计模式、配置项、容错分支。
2. **前后端彻底分离**。后端只暴露 HTTP/SSE 接口;前端只通过 [`01_接口契约.md`](01_接口契约.md) 与后端交互。任何一端都能在另一端尚未完成时独立开发(前端用 mock 层)。
3. **所有提示词使用中文,集中管理**。后端所有 LLM 提示词放在 `backend/app/prompts/` 下的独立 `.md` 文件,通过提示词注册表加载。拷问提示词必须是独立文件,方便单独迭代。
4. **界面不保留任何提示性/说明性文字**,让设计本身说话;风格为「优雅的温暖科技人文」。

## 3. 关键技术决策(已与产品负责人确认)

| 维度 | 决定 | 说明 |
|---|---|---|
| 后端 | Python 3.11+ / FastAPI / Uvicorn | 可直接用 `pdfdeal`(Doc2X)与 OpenAI 兼容 SDK |
| ORM / DB | SQLAlchemy 2.x + SQLite(单文件) | 无账号,匿名会话 |
| 前端 | React 18 + TypeScript + Vite + TailwindCSS | 适配精致界面与流式渲染 |
| LLM 接入 | OpenAI 兼容协议,经 highwayapi 网关,多模型可切换 | 见 §7 环境配置 |
| 拷问对话 | **SSE 流式**(分段协议,见契约 §5) | 逐字呈现问题与手稿 |
| 异步任务 | FastAPI `BackgroundTasks` + asyncio,前端轮询状态 | 不引入 Celery/Redis 等重型队列 |
| PDF 解析 | 仅 Doc2X(`pdfdeal`) | 需 Doc2X API key |
| 训练循环 | 支持同一项目多轮「再次拷打」,旧结果保留可看;不做跨轮进度看板 | |
| 工程形态 | 单仓库,`backend/` 与 `frontend/` 两个独立目录 | 各自独立依赖与启动 |

## 4. 系统模块与数据流

```
┌─────────────┐  文本/PDF   ┌──────────────────────────────────────────┐
│  简历输入页  │ ─────────▶ │ M1 简历解析(Doc2X 转换 + 双 LLM 并行)      │
└─────────────┘            │   产出:简历摘要 + 项目条目[]               │
                           └───────────────┬──────────────────────────┘
                                           │ 解析完成后自动为每个项目
                                           │ 并行触发「预处理」(后台)
                                           ▼
┌─────────────┐           ┌──────────────────────────────────────────┐
│   拷问页     │ ◀──SSE──▶ │ M2 拷问 + 面试官手稿(每项目一个 run)       │
│ 对话 + 手稿  │           │   预处理:初印象 + 首个问题                  │
└─────────────┘           │   对话轮:每轮产出 手稿增量 + 下一问/结束    │
                          └───────────────┬───────────────────────────┘
                                          │ 模型判定结束后自动触发(后台)
                                          ▼
┌─────────────┐           ┌──────────────────────────────────────────┐
│   指南页     │ ◀─轮询──▶ │ M3 总结 / 改进指南                          │
│红黄绿灯+todo │           │   产出:红黄绿灯 + todo[] + 给下一轮的拷打摘要│
│  再次拷打    │           └──────────────────────────────────────────┘
└─────────────┘
```

**三个「后台 + 轮询」节点**(都不阻塞用户操作):
- 简历解析:用户提交后进入解析中,前端轮询直到 `parsed`。
- 项目预处理:解析完成即为**每个**项目并行生成「初印象 + 首问」,用户切到某项目时若已就绪则零等待。
- 改进指南生成:某项目追问结束后自动开始;用户可立刻去拷打别的项目,稍后回来看指南。

**唯一的流式节点**是 M2 的对话轮(用户提交回答 → SSE 逐字返回手稿增量与下一个问题)。

## 6. 模块拆分与可并行性(中等粒度,共 8 个 agent)

| 编号 | 名称 | 目录 | 依赖 |
|---|---|---|---|
| **BE0** | 后端地基与契约 | `backend/app/{main,config,db,models,schemas,llm,pdf}.py` + `prompts/registry.py` | 无(先行) |
| **BE1** | 简历解析模块 | `backend/app/routers/resumes.py` + `services/resume_service.py` + `prompts/resume_*.md` | BE0 |
| **BE2** | 拷问与面试官手稿 | `backend/app/routers/interrogations.py` + `services/interrogation_service.py` + `prompts/interrogation.md` | BE0 |
| **BE3** | 总结与改进指南 | `backend/app/routers/guides.py` + `services/guide_service.py` + `prompts/guide.md` | BE0 |
| **FE0** | 前端地基与设计系统 | `frontend/` 脚手架 + `api/` + `mocks/` + `design/` + 路由 | 无(先行) |
| **FE1** | 简历解析页 | `frontend/src/pages/ResumePage.tsx` + 相关 feature | FE0 |
| **FE2** | 拷问页 | `frontend/src/pages/InterrogationPage.tsx` + 相关 feature | FE0 |
| **FE3** | 改进指南页 | `frontend/src/pages/GuidePage.tsx` + 相关 feature | FE0 |

**两条轨道完全并行**:
- 后端轨道:BE0 先做(规模小、定死共享代码) → BE1 / BE2 / BE3 三者并行。
- 前端轨道:FE0 先做(脚手架 + 设计系统 + API 客户端 + mock 层) → FE1 / FE2 / FE3 三者并行。
- **前端轨道不依赖后端轨道的进度**:FE0 内置 mock 层,前端始终可对着 [`01_接口契约.md`](01_接口契约.md) 独立联调;后端就绪后切掉 mock 即可。

契约 [`01_接口契约.md`](01_接口契约.md) 是两条轨道之间唯一的耦合点,在编码开始前即已冻结。任何一端如需变更契约,必须先改契约文档并通知另一端。

## 7. 环境配置(`.env`)

> **真实密钥不写进任何被跟踪的文件。** `backend/.env`(含真实 `LLM_API_KEY`、`DOC2X_API_KEY`)已由产品负责人预置且被 `.gitignore` 忽略;仓库内只保留占位符 `backend/.env.example`。下方仅展示**字段结构**(占位符),实际值见本地 `backend/.env`。

**后端 `backend/.env.example`(占位符,可提交)**:

```dotenv
# LLM 网关(OpenAI 兼容)
LLM_BASE_URL=https://api.highwayapi.ai/openai
LLM_API_KEY=sk_xxx
# 多模型可切换,逗号分隔;第一个为默认
LLM_MODELS=claude-opus-4-6,gpt-5.5,gemini-2.5-pro
LLM_DEFAULT_MODEL=claude-opus-4-6

# Doc2X(PDF 解析)
DOC2X_API_KEY=sk-xxx

# 应用
DATABASE_URL=sqlite:///./data/app.db
CORS_ORIGINS=http://localhost:5173
```

**前端 `frontend/.env`**(无密钥):

```dotenv
VITE_API_BASE=http://localhost:8000
VITE_USE_MOCK=false   # 并行开发期可设 true,脱离后端独立联调
```

> 安全提示:任何真实密钥只放在 `backend/.env`(已 gitignore);提交前用 `git status` 确认 `.env` 不在暂存区。

## 8. 本地启动

```bash
# 后端(backend/.env 已预置真实密钥,勿用 .env.example 覆盖)
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000

# 前端
cd frontend
npm install
cp .env.example .env
npm run dev            # http://localhost:5173
```

## 9. 全局约定

- **会话标识**:无账号。前端首次访问生成 `session_id`(uuid)存 `localStorage`,所有请求带 `X-Session-Id` 头。后端据此隔离数据。
- **ID**:所有实体主键用字符串 uuid4。
- **时间**:ISO8601 UTC 字符串。
- **错误格式**:统一 `{ "error": { "code": "string", "message": "string" } }`,HTTP 状态码语义化(400/404/409/500)。
- **金句**:面试官手稿与改进指南都是产品的「不可替代功能」,提示词质量是项目成败关键 —— 提示词全文见 [`02_产品提示词附录.md`](02_产品提示词附录.md),不要在编码中临时改写它们。

## 10. v01 明确不做(避免过度设计)

- 不做账号/登录/多端同步。
- 不做跨轮进步可视化看板(仅保留多轮历史可查看)。
- 不做语音、限时压力模式(后续版本)。
- 不做付费/订阅。
- 不做 PDF 本地兜底解析(统一走 Doc2X)。
- 后端不引入消息队列/缓存中间件;后台任务用进程内 `BackgroundTasks` + 数据库状态字段。
